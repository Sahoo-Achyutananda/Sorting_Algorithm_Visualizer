Project Overview
The Sorting Visualizer is a web application that visually demonstrates how common sorting algorithms operate on an array of numbers. The user can:
  1. Generate a new random array.
  2. Change the array size using a slider.
  3. Adjust the speed of the animation.
  4. Visualize different sorting algorithms step-by-step.
The goal of this project was to solidify algorithmic thinking, learn DOM manipulation, and understand asynchronous behavior in JavaScript.
Technology Used
HTML,CSS and JavaScript - nothing else, not even Bootstrap or Tailwind CSS.

1. AbortController()

Discovered how to use AbortController to cancel ongoing asynchronous operations (e.g., stopping sorting when user resets or changes array size).

2. window.onload vs. Reload

Learned that window.onload triggers only when the page is fully loaded for the first time. On reload, it may not work as expected unless combined with the performance API: 

window.addEventListener('load', () => {
    if (performance.getEntriesByType('navigation')[0].type === 'reload') {
        fillContainer();
    }
});

3. sessionStorage vs. localStorage

-> Session Storage stores data per session (cleared when the tab is closed).
-> localStorage persists even after browser or tab is closed.
-> Great for temporarily keeping track of page state.

5. NodeLists vs. Arrays

querySelectorAll() returns a NodeList, not a regular array.
Use Array.from() or spread ([...]) to convert if array methods are needed.

6. CSS Limitation: display is not animatable

Instead of trying to animate display, animate height, opacity, or max-height.

2. Debugging Merge Sort Issues

DOM-based height values were being read in real-time, causing partially updated subarrays to interfere with recursive logic.
Learned that you should never rely on changing DOM values during recursion‚Äîwork only on your own array copy.
